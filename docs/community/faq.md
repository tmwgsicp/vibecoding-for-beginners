# 常见问题 (FAQ)

---

## 📖 学习相关

### Q: 完全不会编程,真的能学吗?

**A:** 能。Vibe Coding的核心是方法论,不是语法。

**证明:**
- AI会帮你写代码
- 你负责思考"要做什么"
- 不需要记语法,只需要理解逻辑
- 很多人都是零基础开始的

**但需要:**
- 逻辑思维能力
- 学习意愿
- 坚持和耐心

---

### Q: 需要多长时间?

**A:** 取决于学习方式

**系统学习:** 4周
- Week 1: 学习历程 + 方法论
- Week 2: 真实案例
- Week 3-4: 动手实践

**快速上手:** 1周
- 只看核心方法论
- 立即动手做项目

**深度学习:** 2-3个月
- 完整学习所有内容
- 做多个项目

**关键:** 每周投入5-10小时

---

### Q: 学完能做什么?

**A:** 能独立做SaaS产品

**具体包括:**
- 从idea到MVP
- 前端+后端+数据库
- 用户系统
- 支付系统
- 部署上线
- 商业化运营

---

### Q: 和传统编程教程有什么区别?

**A:** 完全不同的思路

| 传统教程 | Vibe Coding |
|---------|-------------|
| 教语法 | 教方法论 |
| 从基础开始 | 从做产品开始 |
| 理论 + 练习 | 真实案例 + 实战 |
| 学完不知道做什么 | 学完就能做产品 |
| 需要几个月打基础 | 第一周就能做出东西 |

**核心差异:** 
- 传统教程让你"成为程序员"
- Vibe Coding让你"用AI做产品"

## 🤖 AI工具相关

### Q: AI工具需要付费吗?

**A:** 认真做项目需要付费

**能访问国外(推荐):**
- Cursor Pro($20/月) + Claude Code
- 包含 Claude Sonnet 4.6 使用额度

**无法访问国外:**
- 通义灵码/Trae(IDE 基础功能免费)
- MiniMax/Kimi/GLM(需购买 API 使用)
- 不是零成本,只是网络受限的替代

**学习阶段:**
- 可以先用免费额度试用
- 认真做项目就要升级付费

---

### Q: Cursor 和 Claude Code 怎么用?

**A:** 两个都用 Claude,灵活选择

- Cursor Pro: 可视化 IDE,新手更友好
- Claude Code: CLI 终端工具,有学习门槛
- 可以多开并行开发
- 根据任务特点选择工具

**建议:** 新手从 Cursor 开始。

---

### Q: 不会提示词怎么办?

**A:** 不需要复杂的提示词技巧

**核心原则:**
1. 说清楚要做什么
2. 提供上下文
3. 明确输入输出
4. 让AI解释代码

**例子:**

❌ **不好的提问:**
```
写个爬虫
```

✅ **好的提问:**
```
我在做微信RSS项目,架构是Python+FastAPI。
现在需要实现爬虫模块。

要求:
- 输入: URL
- 输出: HTML内容
- 需要: 异步、代理池、重试机制

请给代码。
```

[更多 AI 协作技巧 →](/guide/05-tools/cursor-skills)

## 🚀 项目相关

### Q: 应该从零做还是二开?

**A:** 取决于情况

**从零做 - 适合:**
- 有独特的想法
- 想完全掌控
- 学习目的

**二开/迁移 - 适合:**
- 在现有项目基础上改进
- 想快速上线
- 降低风险

**实战建议:**
- 第一个项目: 跟着教程做(降低难度)
- 第二个项目: 二开一个GitHub项目
- 第三个项目: 从零做自己的idea

## 🛠️ 技术相关

### Q: Python和JavaScript选哪个?

**A:** 看项目需求

**选Python - 如果:**
- 数据处理
- 爬虫
- AI/ML相关
- 后端API

**选JavaScript - 如果:**
- 前端界面
- 全栈开发
- 实时通信

**我的建议:**
- 不知道做什么 → Python(简单、用途广)
- 做网页 → JavaScript(前后端都能用)

**都可以:** AI都会写,不用太纠结

[详细对比 →](/guide/01-methodology/05-tech-decisions.md)

---

### Q: 不懂技术术语怎么办?

**A:** 查工具和概念文档

**遇到不懂的:**
1. 先查 [工具和概念](/guide/05-tools/)
2. 还不懂就问AI
3. 记录下来,下次就懂了

**不要:**
- 背诵所有术语
- 深入研究每个技术
- 因为不懂就放弃

**要:**
- 知道"是什么"就够了
- 用到再深入了解
- 积累是个过程

### Q: 部署上线很难吗?

**A:** 第一次会遇到问题,但有完整指南

**常见困难:**
- 域名、DNS配置
- HTTPS证书
- 防火墙设置

**解决方案:**
- 跟着 [部署指南](/guide/01-methodology/12-deployment-guide.md)
- 一步步操作
- 遇到问题查文档
- 还不行就在社区问

**第一次:** 2-3小时  
**熟练后:** 30分钟
