# AI 编程工具选择指南

---

## 直接看结论

**能访问国外:**
- IDE: Cursor Pro($20/月)
- CLI: Claude Code(官方最强工具)

**无法访问国外:**
- IDE: 通义灵码 或 Trae
- 模型: MiniMax/Kimi/GLM(需付费)

别花太多时间对比,选完开始做项目。

## 💻 IDE 选择

### Cursor - 付费 IDE 唯一推荐

**官网:** https://cursor.com/

我为什么只推荐这一个?

不是因为其他 IDE 不好,是因为:
1. Composer 2 多文件编辑是独家的
2. Agent Mode 自动化测试部署省太多时间
3. $20/月包含 Claude 额度,性价比最高
4. 其他付费工具要么能力不足,要么更贵

**定价:**
- Pro($20/月): 个人用,推荐
- Teams($40/用户/月): 团队协作

---

### 通义灵码 - 国内开发者选择

**官网:** https://tongyi.aliyun.com/lingma/

**什么时候选:**
- 无法访问国外网络
- 企业内网开发

**核心功能:**
- 编程智能体(类似 Cursor 的 Agent)
- 多文件编辑
- 支持 VSCode/JetBrains/Visual Studio
- 200+ 编程语言

**注意:** 基础功能免费,但认真做项目需要付费调用模型 API。

---

### Trae - 字节跳动出品

**官网:** https://trae.cn

**什么时候选:**
- 无法访问国外
- 需要快速做原型

**核心功能:**
- SOLO Builder: 从 0 到 1,快速搭建
- SOLO Coder: 从 1 到 N,迭代优化
- Figma 导入转代码
- 多模态输入(截图、草图)
- 一键部署

---

## 🔧 终端工具

### Claude Code - Anthropic 官方 CLI

**官网:** https://code.claude.com/

**重要:** 这是 Anthropic 官方推出的终端工具,不是 IDE 但能力顶级。

**为什么要用 CLI?**

很多人以为 AI 编程只能在 IDE 里用,其实终端工具更适合某些场景:
- Git 工作流自动化: `git diff | claude code review`
- 快速写脚本: 不用打开 IDE
- 日志分析: `tail app.log | claude analyze`
- 自动化任务: 结合 Shell 脚本

**安装:**
```bash
# macOS/Linux/WSL
curl -fsSL https://claude.ai/install.sh | bash

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex
```

**实际使用场景:**
- Git 工作流自动化
- 快速写脚本和工具
- 多开终端并行处理任务
- 配合 Cursor 一起用,灵活选择

**模型:**
- 使用 Claude Opus 4.6(官方最强模型)
- 独立计费,不和 Cursor 共享额度
- 成本高但质量也是最好的

## 🤖 模型选择

### 我的实际选择

**主力:** Claude(通过 Cursor 或 Claude Code)  
**备用:** 国内模型(网络问题时用)

为什么这样选?
- 直接选能力最强的,不纠结
- 真遇到限额或网络问题才考虑其他的
- 时间比钱贵

---

### 国外模型

**Claude Opus 4.6** - 能力最强
- 推理能力、架构设计最好
- Claude Code 默认使用这个
- 成本高(比 Sonnet 贵 5 倍)
- 关键功能、架构决策时用

**Claude Sonnet 4.6** - 性价比高
- 编程能力也很强(略弱于 Opus)
- Cursor Pro 包含使用额度
- 日常开发够用
- 成本相对低

**怎么选:**
- 重要功能、架构设计 → Opus
- 日常开发、快速迭代 → Sonnet
- 预算紧张全用 Sonnet 也够

---

### 国内模型

**重要说明:**

这不是"省钱方案",是网络受限时的替代。

认真做项目,国内模型一样要付费调用 API。

**MiniMax M2.7**
- 编程能力强
- 全栈开发都能用
- API 付费使用

**Kimi K2.5**
- 视觉编码(截图→代码)
- 超长上下文(处理大文件)
- API 付费使用

**GLM-5.1**
- 国产顶级模型
- 中文支持最好
- API 付费使用

**怎么选:**
- 三个都试试,选自己用着顺手的
- 我没有特别偏好,能力都够用
- 别花时间纠结

## 📊 实际使用场景

### 能访问国外(推荐)

**工具:**
- IDE: Cursor Pro($20/月)
- CLI: Claude Code
- 模型: Opus(关键) + Sonnet(日常)

**为什么这样搭配:**
- 这是目前能力最强的组合
- 成本 $20-50/月,但效率提升 5-10 倍
- 做出第一个产品后完全能回本

---

### 无法访问国外

**工具:**
- IDE: 通义灵码 或 Trae
- 模型: MiniMax/Kimi/GLM 任选一个

**成本:**
- IDE 基础功能免费
- 模型 API 按调用量付费
- 比国外便宜,但不是零成本

---

### 前端/UI 开发

**推荐:** Kimi K2.5

**为什么:**
- 视觉编码能力强
- 截图直接转代码
- 处理 UI 调整特别快

**搭配:**
- Cursor + Kimi 或 通义灵码 + Kimi

---

**不要:**

## ❓ 常见问题

### 新手应该选哪个?

**能上国外:** Cursor Pro + Claude Code

**网络受限:** 通义灵码/Trae + 国内模型

学习阶段可以用免费额度,认真做项目就要付费。

---

### Cursor 值得花钱吗?

**我的经验:**

值得,非常值得。

$20/月提升效率 5-10 倍,做出第一个产品就能回本。

不要在工具上省钱,时间更贵。

---

### 国内工具需要付费吗?

**实际情况:**

IDE 基础功能免费,但模型 API 调用要付费。

不是"省钱方案",只是网络受限时的替代。

认真做项目,国内外都一样要投入。

---

### 模型怎么选?

**能上国外:** Claude(Opus 优先,Sonnet 够用)

**网络受限:** 国内三个任选一个

**建议:** 别花时间对比,都试试选顺手的。

---

### Claude Code 和 Cursor 有什么区别?

**区别:**

- Cursor: IDE,有可视化界面,新手更直观
- Claude Code: CLI 终端工具,有学习门槛

**实际使用:**

不是固定比例,看项目和习惯:
- 新手建议先用 Cursor(可视化界面更友好)
- 熟练后可以两个都用
- 可以多开窗口并行开发(Cursor 开多个项目,Claude Code 开多个终端)
- 直接用 Claude Code 终端也很高效

**建议:**

新手从 Cursor 开始,熟练后根据需要选择工具。
