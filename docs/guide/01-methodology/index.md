## 为什么方法论最重要？

**方法论 ≠ 教你写代码**

方法论教的是:
- 如何思考问题
- 如何做决策
- 如何与AI协作
- 如何解决问题

```
没有方法论:
遇到问题 → 不知道怎么办 → Google乱搜 → 试了很多方法 → 还是不行 → 放弃

有方法论:
遇到问题 → 分析问题类型 → 查对应章节 → 找到解决思路 → 快速解决 → 继续前进
```

**简单说:** 方法论是做产品的地图。

## 怎么学？

### 建议: 按顺序系统学习

全部 14 章已完成,建议按顺序学习:
- 1-3章: 问题发现与验证
- 4-7章: 技术决策与 AI 协作
- 8-14章: 理解代码、测试、架构、部署、问题解决、Git、迭代

**时间:** 3-5 天快速浏览,1-2 周深度理解

---

### 或者: 边做边查

不需要背下来。做项目时:
- 不知道怎么开始 → 看第 1-3 章
- 不知道选什么技术 → 看第 4-5 章
- 不知道怎么和 AI 对话 → 看第 7 章
- 遇到部署问题 → 看第 11 章
- 项目上线后出问题 → 看第 12 章

**方法论是工具书,不是教科书。**
